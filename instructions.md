# IoT-Based Livestock Health Monitoring System – User Interface Design Prompt

## Overview
Design a simple, intuitive, and role-based user interface for an IoT-based livestock disease prevention and loss reduction system. The system has two main users:
1. Livestock Farmers
2. Ministry of Livestock and Fisheries Officials

The interface must prioritize usability in low-resource and rural environments, support mobile and web access, and present health data clearly for decision-making.

---

## User Roles and Access Control
- Implement role-based access:
  - Farmer: Access to individual livestock data.
  - Ministry Official: Access to aggregated and anonymized regional data.
- Secure login using username and password.

---

## 1. Farmer User Interface

### Target Platform
- Mobile-first web interface (responsive design)
- Must work on low-end smartphones and basic browsers

### Main Dashboard Features
- Display list of registered livestock with:
  - Animal ID
  - Current health status (Normal / Warning / Alert)
- Visual indicators (icons or colors) for abnormal conditions

### Livestock Details Page
- Show detailed data for a selected animal:
  - Body temperature trends
  - Activity and movement levels
  - Rumination proxy indicators
  - Environmental conditions
- Historical data displayed using simple line charts

### Alerts & Notifications
- Display recent alerts generated by the system
- Clearly explain the reason for each alert
- Indicate whether an SMS alert has been sent

### Recommendations Section
- Provide basic guidance such as:
  - “Check animal”
  - “Isolate animal”
  - “Contact veterinary officer”

### Design Considerations
- Simple language
- Minimal text
- Large buttons and icons
- Offline-friendly UI where possible

---

## 2. Ministry of Livestock and Fisheries Interface

### Target Platform
- Web-based dashboard for desktop and tablet use

### Main Dashboard Features
- Overview of livestock health at:
  - District level
  - Regional level
- Statistics including:
  - Number of monitored animals
  - Number of alerts detected
  - Disease risk trends over time

### Data Visualization
- Charts and graphs showing:
  - Disease occurrence trends
  - Regional health comparisons
- Heat maps to highlight high-risk areas

### Reporting & Analysis
- Generate summary reports (daily, weekly, monthly)
- Export data in CSV or PDF format

### Data Privacy
- Do NOT show individual farmer names or exact animal locations
- Display only aggregated and anonymized data

---

## General UI Requirements
- Clean and minimal design
- Low bandwidth optimization
- Fast loading times
- Consistent color scheme
- Clear navigation between sections

---

## Goal of the Interface
The user interface should support early disease detection, improve livestock health management, reduce losses for farmers, and enable effective disease surveillance and policy planning for the Ministry of Livestock and Fisheries.
